## Goal
将大体积 `log_details` 请求体/响应体改为可选上传到腾讯云 COS，数据库只保存引用和元数据，小体积内容继续内联保存。

## Steps
- [x] 梳理当前 `log_details` 写入、缓存、读取、清理路径
- [x] 增加 COS 存储配置和上传/下载实现
- [x] 扩展 `LogDetail` 元数据字段并接入写入/读取
- [x] 确保旧日志清理不删除 COS 对象
- [x] 运行后端测试
- [ ] 提交并推送

## Decisions
- COS 默认关闭，仅 `LOG_DETAIL_STORAGE_TYPE=cos` 且配置完整时启用。
- COS 对象永远不删除；旧日志清理只删除 Redis 和数据库记录。
- 保留旧内联数据兼容，读取时对 COS 引用透明下载。

## Goal
新增抽奖兑换码：管理员可创建同一兑换码供多个用户使用，每个用户限兑换一次，奖励额度支持区间随机或指定额度列表随机。

## Steps
- [x] 梳理现有兑换码模型、接口、前端管理页和用户兑换页
- [x] 设计兼容现有兑换码的抽奖字段和用户兑换记录
- [x] 实现后端创建/更新/兑换/列表展示逻辑
- [x] 实现前端管理员配置入口和列表显示
- [x] 补充测试并运行后端/前端校验

## Decisions
- 保留普通兑换码现有行为，抽奖兑换码作为新模式扩展。
- 抽奖兑换码按用户维度限制一次，不按兑换码全局一次。
- 抽奖兑换码支持管理员指定密钥；未指定时仍自动生成。
- 固定额度随机使用 `额度:权重` 表示概率，未写权重时默认权重为 1。
- `max_redeem_count = 0` 表示不限制总领取次数，大于 0 时达到数量后停止领取。
